runTransaction(firestore, updateFunction)
Executes the given updateFunction and then attempts to commit the changes applied within the transaction. 
If any document read within the transaction has changed, the updateFunction will be retried.

When the match button is clicked or pressed:
- A first transaction occurs
- If the user's matchedID blank (nobody matched with them), set their looking flag to 1
- Else, pass the data to match and set looking flag to 0

The first transaction sets the user's looking flag to 1

- A second transaction occurs
- If the user's looking flag is 1 and their matchedID is blank (if the user has not been matched),
  then iterate through list of users for a match
  pass data for client side matching after transaction ends
  exit loop
- else
  pass data for client side matching after transaction ends
