runTransaction(firestore, updateFunction)
Executes the given updateFunction and then attempts to commit the changes applied within the transaction. 
If any document read within the transaction has changed, the updateFunction will be retried.

When the match button is clicked or pressed:
- A first transaction occurs
- If the user's matchedID blank (nobody matched with them), set their looking flag to 1
- Else, pass the data to match and set looking flag to 0

The first transaction sets the user's looking flag to 1

- A second transaction occurs
- If the user's looking flag is 1 and their matchedID is blank (if the user has not been matched),
  then iterate through list of users for a match
  pass data for client side matching after transaction ends
  exit loop
- else
  pass data for client side matching after transaction ends

Case 2: A joins Q alone, B joins Q later, B's transactions do not close before A returns:

t = 2 |A-Transaction 2:
    *B clicks onFindMatch at this time
    Iterates through all users and exits (can pass data to signify this)
t = 3 |A-Transaction 2 manual retry:
    Since the transaction exited without finding a match, appropriate data is returned client-side
    This creates a client-side instigated retry a finite amount of times before displaying a prompt to the user
    If B's transaction 2 is complete before this transaction starts:
        A's transaction 2 jumps to else clause and passes appropriate data
    If B's transaction 2 is not complete before this transaction starts, but their transaction 1 is complete:
        A's transaction 2 goes through the if clause
        Whichever user completes their transaction first dictates what occurs next
        If A's transaction 2 finishes first, then B's will restart and catch up to A who is already matched with B
        If B's transaction 2 finishes first, then A's will restart and catch up to B who is already matched with A

In other words, the transaction that finishes later will restart and then instantly be matched with whoever signed them.
